How It Works

Understanding the Workflow of PATHRIKA

Step 1: Login and Role-Based Authentication

Users log in with their credentials. Based on their role (Admin, Teacher, or Superintendent), they are granted specific permissions.

Step 2: Admin Requests Question Papers

The Admin sends a request to Teachers to prepare question papers for specific subjects and schedules.

Step 3: Teachers Upload Question Papers

Teachers receive the requests and upload the question papers in PDF format, securely stored on IPFS (InterPlanetary File System).

Step 4: Hash Code Generation and Blockchain Storage

A unique hash code is generated for each uploaded question paper and stored on the blockchain, ensuring tamper-proofing and security.

Step 5: Time-Based Access for Superintendents

The Superintendent can request access to the question papers, but the system reveals the hash code only after the pre-defined schedule.

Step 6: Secure Retrieval of Question Papers

The Superintendent uses the hash code to securely retrieve the question papers from IPFS, ensuring integrity and confidentiality.
